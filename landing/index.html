HOW IT
WORKS

01
Data Collection
ESP32 Reads All Sensors
The ESP32 microcontroller continuously polls every sensor: the DHT11 reads machine temperature every 2 seconds, the I2S MEMS microphone captures sound RMS every second, the VL53L0X ToF sensor measures distance for motion detection, the IR sensor counts people crossing a zone, and the emergency button is debounce-monitored in real time.
DHT11 I2S MIC VL53L0X IR BTN
02
Threshold Logic
Machine Temperature Monitoring
The system continuously monitors machine temperature using the ESP32 and applies a threshold-based evaluation mechanism to ensure operational safety.

Temperature Thresholds:
The system uses three configurable thresholds to detect different levels of temperature anomalies:

Safe Threshold: 30°C (Normal operation)
Warning Threshold: 40°C (Requires attention)
Danger Threshold: 50°C (Immediate action required)

Real-Time Monitoring:
The ESP32 reads temperature every 2 seconds and compares it against these thresholds, updating the system state accordingly.

System Purpose:

Prevent machine overheating
Provide early warnings for potential failures
Ensure safe operating conditions
30°C SAFE 40°C WARNING 50°C DANGER


03
Sound Monitoring
Machine Sound Analysis
The system continuously monitors machine sound levels using an I2S MEMS microphone connected to the ESP32 and evaluates noise conditions in real time.

Real-Time Detection (ESP32):
The ESP32 captures audio signals and calculates the RMS (Root Mean Square) value to determine sound intensity and detect abnormal noise patterns.

Threshold Evaluation (On-Device):
The system compares the RMS value with a predefined threshold to identify abnormal conditions.

System Response:

When abnormal sound is detected, the buzzer is activated
The emergency flag is automatically set to true
Sensor data is instantly updated to the cloud database

Dashboard Visualization:
The dashboard displays sound levels in real time and classifies them into:

🟢 Normal: Sound within safe range
🟡 Warning: Slight increase in noise
🔴 Critical: Abnormal or high sound detected

This helps detect unusual machine behavior, potential faults, and unsafe operating conditions.
RMS ≤ 25 → NORMAL RMS > 25 → ABNORMAL HIGH RMS → BUZZER ON ABNORMAL → EMERGENCY


04
Human Detection
Restricted Area Monitoring
The system detects human presence in restricted zones using an IR sensor connected to the ESP32 and maintains a real-time count of detected entries.

Real-Time Detection (ESP32):
The IR sensor detects objects based on infrared reflection. When a person crosses the sensor (LOW signal), it is identified as an entry event.

Event Handling:

Each detection increments the people count
A short buzzer beep is triggered for every entry
Debounce logic ensures accurate counting without false triggers

Data Transmission:
The updated people count is continuously stored in the cloud database and persists even after system reboot.

Dashboard Visualization:
The dashboard displays the number of people detected and allows monitoring of restricted area access in real time.

System Purpose:

Monitor unauthorized access
Track movement in restricted zones
Maintain safety compliance in industrial environments


05
Emergency Control
Emergency Trigger
The system includes a physical emergency button connected to the ESP32 that allows immediate manual intervention during critical situations.

Real-Time Detection (ESP32):
The emergency button is continuously monitored using debounce logic to ensure accurate detection of presses without noise or false triggers.

Trigger Mechanism:

When the button is pressed, the system instantly sets the emergency state to true
The emergency status is immediately sent to the cloud database
This action overrides all normal system operations

System Response:

Emergency alert is activated across the system
Buzzer and safety mechanisms can be triggered
Critical status is displayed on the dashboard in real time

Dashboard Control:
The dashboard provides an option to reset the emergency state, which updates the database and restores normal system operation.

System Purpose:

Provide instant manual shutdown capability
Ensure operator safety during hazardous conditions
Allow quick response to unexpected emergencies


06
Cloud Sync
Firebase Realtime Database
The system uses Firebase Realtime Database to store and synchronize all sensor data in real time between the ESP32 device and the web dashboard.

Real-Time Data Sync:
The ESP32 continuously sends sensor readings to Firebase using the Firebase ESP Client library, ensuring low-latency updates and live monitoring.

Data Structure:
All data is organized under the /sensors/ path, including:

machineTemp → Machine temperature readings
soundLevel → Sound RMS values
motion → Machine movement status
people → Total people detected
emergencyBtn → Emergency state flag

Authentication & Security:
Secure access is handled using email and password authentication, ensuring only authorized devices and users can read or write data.

Data Persistence:
Critical values such as people count and emergency state are stored persistently, allowing data recovery even after system reboot or reconnection.

System Purpose:

Enable real-time monitoring across devices
Maintain synchronized system state
Provide reliable cloud-based data storage
REAL-TIME SYNC /sensors/ PATH EMAIL AUTH PERSISTENT DATA


07
Live Dashboard
Real-Time Web Monitoring Interface
The web dashboard provides a real-time monitoring interface by subscribing to Firebase Realtime Database using ref.on('value'), enabling instant data updates without the need for polling.

Real-Time Updates:
Every change in the database is pushed immediately to the UI, ensuring live synchronization between the ESP32 and the dashboard.

Dynamic UI Behavior:

Sensor cards update instantly with new values
Status banner changes color based on system condition
Sparklines and charts redraw dynamically
Alerts are generated and logged in the history panel

Analytics & Visualization:
The dashboard uses Chart.js to process and visualize data trends, helping users understand system behavior over time.

System Purpose:

Provide real-time visibility of all sensor data
Enable quick identification of abnormal conditions
Maintain a responsive and interactive monitoring experience

Sensors

WHAT'S
MONITORED

🌡️
Machine Temperature
DHT11 sensor reads ambient machine temperature every 2 seconds and pushes to Firebase. Colour-coded sparkline tracks trends over the last 20 readings.
≤35°C — NORMAL ≤40°C — WARNING >40°C — CRITICAL
🎙️
Sound Level (RMS)
MEMS I2S microphone captures audio at 44.1kHz. RMS amplitude is computed on-device every second. Abnormal sound triggers buzzer and emergency flag instantly.
≤25 RMS — NORMAL >25 RMS — ABNORMAL
📡
Machine Motion
VL53L0X time-of-flight sensor measures distance to detect if the machine has shifted position. Distances over 30cm flag a HIGH motion state.
≤30cm — LOW >30cm — HIGH
🚶
People / Zone Access
IR sensor detects each person entering a restricted zone. Count is cumulative and persists across ESP32 reboots via Firebase. A beep fires on every detection. Reset is available from the dashboard.
0 — CLEAR >0 — DETECTED
🔴
Emergency Stop Button
Debounced physical button on GPIO 4. A press sets emergencyBtn=true in Firebase, triggering full dashboard emergency mode. Can also be triggered remotely or by sensor thresholds.
false — STANDBY true — TRIGGERED
⚡
Auto-Trigger Logic
The system self-activates emergency mode when temperature exceeds 50°C or sound is abnormal — even without a button press. The ESP32 writes the emergency flag automatically to Firebase.
TEMP > 50°C → AUTO STOP SOUND > 25 → AUTO STOP

Hardware

THE
HARDWARE

DHT11 Temperature Sensor
Reads ambient machine temperature. Polled every 2s. Thresholds at 35°C, 40°C, 50°C.
MEMS Microphone (I2S)
WS:25, BCLK:26, SD:33. 44.1kHz sample rate. RMS computed over 1024-sample DMA buffer.
VL53L0X ToF Distance Sensor
Measures distance via time-of-flight. >30cm = HIGH motion status. Detects machine positional shift.
IR Sensor (People Counter)
Active-low output. Debounced at 80ms. Falling edge = person detected. Count persists in Firebase.
Emergency Stop Button
INPUT_PULLUP with 50ms debounce. LOW press sets emergency = true in Firebase immediately.
Buzzer
Fires on abnormal sound detection and on each IR person count. Controlled bidirectionally via Firebase.
OLED Display
Displays real-time system data locally on the device, including temperature, sound level, movement status, and emergency state. Provides instant visual feedback without requiring access to the web dashboard. Automatically updates values and highlights critical conditions for quick on-site monitoring.
